Project plan

Overview – Oxygen Factory

This project aims to build an organic, interactive, site based experience which takes user inputs ( from their behaviour in the space) in order to create different experiential outcomes. By exploring the ideas of climate change, pollution, sickness and world resource use, the installation would offer different visual and auditory responses to the user as they are in the space.

Motivation

There are a number of people and artists whose work embrace technology and code to look at the interface between the living world and the digital/ electronic world. Some examples of which are :

  • United Visual Artists (https://www.uva.co.uk/)
  • Random International (https://www.random-international.com/work)

My aim would be to expand my current practice to incorporate some of this inspiration, understanding that a lot of coding experience is required in order to complete the technical requirements of the project.

Description.

In the space there are multiple screens ( projections on a surface or physical LED screens) which project visual results ( output) base on the large amount of recorded data of people within the space. In the centre of the room is a large tree like structure made up of individual, programmable LED lights, which changes its output( colour and intensity) based on the same input information.


At this stage Arudino sensors would be used to :

  • record the users interaction with multiple light sensors (detecting movement, the number of people in a particular area or the wholes space)
  • record or detect audio – used to trigger and audio recording pfor manipulation and play back in real time in the installation space
  • record video – used for manipulation and playback across the screens in the installation space – a feedback loop of live information.


From an artist/ user perspective, the project’s aim is to create an environment like the lungs/ a large tree, however to explore different interactions with them ( eg how does pollution affect the space? How does disease affect how we breathe?)

From an IT perspective, the challenges are the writing of sophisticated programs which are able to react or respond dynamically and instantaneously to changes in the space. The potential to be working with multiple user inputs means that a system, or systems, need to be designed to deal with contradictory information ( and prioritise it) as well as produce real time changes to the physical envrironment.

Skills Required

There are a lot of skills required for this project, including :

  1. Artist and design – Desining the final installation ( how it looks) incorporating all of the inputs and outputs and relevant hardware.
  2. Construction – the envisioned space is quite large, and a knowledge of how to build the item safely and securely as a key factor
  3. Hardware and Electronics – safely wiring multiple electronic sources ( chips, lights, screens, AV recorders) will require a detailed knowledge of electronic wiring and/ or electrical engineering
  4. Software – there are multiple programs which will need to be written so that the hardware responds as desired to the provided inputs. These will be running simultaneously and could be with:
    • One mega program incorporating all processes for the installation ( risky) or
    • Multiple programs running on multiple computers with separate routing and wiring for each system.

This project will involve multiple people working together on connected portions of the project, as all aspects interact with each other ( software, hardware, construction, design).

Outcome.

The primary outcome would be the creation of an interactive audio-visual environment that adapts to the movement of the user/s and (hopefully) looks good. In completing this , however the main solution will involve devising a system, or systems, that can take inputs from a large number of sources in real time, and produce a changing stream of output based on this through multiple and varied output channels ( screens, LED lights, audio speakers, video screens). The flexibility, responsiveness, and adaptability of the output needs to be explored further, of course, but a well-designed project will have solve all of these questions.
